# REFACTORING

## Low-level refactoring : 변수와 함수를 변경하는 것

- 변수, 함수 이름은 사람이 이해하기 쉽도록 의미있고, 구체적으로 작성
  - 이름이 너~무 긴 것도 좋진 않음
- 함수 : 프로그램을 이루는 가장 기본적인 단위, 프로그램을 작은 단위로 나누는 수단<br>
  재사용 할 수 있도록, 가독성 좋게!<br>
  함수 이름만 보고도 어떤 역할을 하는지 알 수 있도록! ➡️ 세부 구현 사항을 일일이 확인하지 않아도 괜찮도록

  1. 코드가 어떤 일을 하는지 파악
  2. 독립적인 함수로 추출
  3. 목적에 맞는 이름 짓기 (ex. displayUser())

- 지역변수는 사용하는 곳과 가장 가까운 곳에 정의

### 이름 짓기 포인트

- 변수 : 무엇을 담고 있는지 잘 나타내야 함! / 보통 명사로
- 함수 : 무엇을 하는지 잘 나타내야 함! / 보통 동사로(displayBanner, getTotalPrice), 명령법, 단언적으로
  - getter를 가지고 있는 함수인 경우 get 생략 가능
  - 네트워크상에서 가져올 때 : fetch
  - 매개변수, 인자는 어떤 데이터가 필요한지 잘 나타낼 수 있도록

### 함수의 매개변수 포인트!

- 매개변수의 갯수는 최대 3개를 넘지 않는 것이 좋다
- 매개변수가 많이 필요한 경우라면, 연관있는 데이터 구조 하나로 묶어주기(자료구조, 클래스 등)

## High-level refactoring : 모듈/클래스와 관련된 리팩토링

## 캡슐화 : 내부 구현 사항을 숨기고, 외부에서 **_필요한 사항들만 공개_**

- 모듈, 클래스와 같은 형태로 캡슐화!
